# GitHubにSSH接続して、Git操作を行う手順
## Step 1. SSH用フォルダに移動する
まず、SSHの設定ファイルを置くフォルダに移動します。
`cd ~(自分のディレクトリ）/.ssh`

もしフォルダが存在しない場合は、新しく作成します。
` mkdir ~/.ssh`
`cd ~/.ssh`

## Step 2. SSH鍵を作成する
SSH鍵（GitHubと安全に通信するための鍵）を作成します。
`ssh-keygen -t rsa`

途中でパスワードを聞かれますが、
設定しない場合はすべて Enter を押してください。


## Step 3. 鍵ファイルが作成されたか確認する
ファイル一覧を表示します。
`ls`

以下の2つのファイルが表示されていれば成功です。
id_rsa（秘密鍵）
id_rsa.pub（公開鍵）

## Step 4. 公開鍵の中身を表示する
GitHubに登録するため、公開鍵の中身を表示します。
`cat id_rsa.pub`
表示された 文字列すべて をコピーします。

## Step 5. 公開鍵をGitHubに登録する
コピーした公開鍵を、GitHubのSSH Key設定画面に貼り付けます。

## Step 6. GitHubとの接続を確認する
GitHubに正しく接続できるか確認します。
`ssh -T git@github.com`

以下のようなメッセージが表示されれば接続成功です。接続完了
Hi {アカウント名}! You've successfully authenticated, but GitHub does not provide shell access.

# Gitの基本操作
## Step 7. ファイルを1つだけステージに追加する
特定のファイルだけをステージ（コミット前の準備状態）に追加します。
`git add ./html_CSS_1/01.html`

## Step 8. 変更したファイルをすべてステージに追加する
作業したファイルをまとめてステージに追加します。
`git add -A`

## Step 9. コミットする
ステージに追加した内容を、履歴として保存します。
`git commit -m '{コミットメッセージ}'` 
例：
`git commit -m "HTMLの初期構造を追加"`

## Step 10. リモートリポジトリに反映する
GitHub（リモートリポジトリ）に変更を反映します。
`git push`
